# plan-then-diff

この skill は、実装前の計画提示と差分管理を強制する。

## 必須手順

1. 関連する既存コードを確認する。
2. 影響範囲を特定する（contracts / api / bot / docs）。
3. 2〜6ステップの簡潔な作業計画を提示する。
4. 計画に沿って実装する。
5. 実行したコマンドと結果を報告する。

## 必須出力形式

- 変更概要
- 影響範囲
- 実行したコマンドと結果
- 残対応事項（あれば）
